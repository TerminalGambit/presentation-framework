Presentation Framework

Generate branded HTML slide decks from YAML configuration and JSON metrics data. Fully themeable, AI-ready, and shareable as a single zip.

6 Layouts

Title, two-column, three-column, data-table, stat-grid, and closing — cover every presentation need.

Data Pipeline

All values flow from metrics.json via {{ metrics.x.y }} interpolation. No hardcoded data.

Custom Themes

Set primary and accent hex colors in YAML — all CSS variants are derived automatically.

CLI Tooling

pf init, pf build, pf serve, pf zip — scaffold, build, preview, and share.

Navigator

Keyboard navigation (←→), grid overview (G), fullscreen (F), progress bar — built in.

AI-Ready

SKILL.md teaches AI agents to create presentations autonomously from a metrics file.

Installation

pip install -e /path/to/presentation-framework

Requirements: Python 3.10+, Jinja2, PyYAML, Click. All installed automatically.

Verify with:

pf --help

Quick Start

# 1. Scaffold
pf init my-deck

# 2. Edit my-deck/presentation.yaml and my-deck/metrics.json

# 3. Build and open in browser
pf build --config my-deck/presentation.yaml --metrics my-deck/metrics.json --output my-deck/slides --open

# 4. Share
pf zip --dir my-deck/slides

pf init

pf init <name>

Creates a new presentation project directory with starter files:

<name>/
├── presentation.yaml   # Starter deck config
├── metrics.json        # Empty metrics template
└── slides/             # Output directory (empty)

pf build

pf build [OPTIONS]
OptionDefaultDescription
-c, --configpresentation.yamlPath to YAML config
-m, --metricsmetrics.jsonPath to metrics JSON
-o, --outputslidesOutput directory
--openoffAuto-open present.html in browser

The build pipeline:

  1. Loads YAML config and JSON metrics
  2. Resolves {{ metrics.x.y }} interpolation in all slide data strings
  3. Renders each slide using its layout template (Jinja2)
  4. Renders the navigator shell (present.html)
  5. Copies theme CSS and generates custom variables.css from theme config

pf serve

pf serve [OPTIONS]
OptionDefaultDescription
-d, --dirslidesDirectory to serve
-p, --port8080Port number

Starts a local HTTP server. Open http://localhost:8080/present.html to present.

pf zip

pf zip [OPTIONS]
OptionDefaultDescription
-d, --dirslidesSlides directory to package
-o, --output<dir>.zipOutput zip file path

Packages the built slides into a self-contained .zip file. Recipients extract and open present.html — no server needed.

presentation.yaml

The YAML config has three top-level sections:

meta:
  title: "Deck Title"
  authors: ["Author 1", "Author 2"]
  institution: "School or Company"
  date: "2026-01-01"

theme:
  primary: "#1C2537"        # Background color (any hex)
  accent: "#C4A962"         # Accent color (any hex)
  fonts:
    heading: "Playfair Display"
    subheading: "Montserrat"
    body: "Lato"

slides:
  - layout: title
    data:
      title: "My Title"

metrics.json

All dynamic data lives in the metrics file. Reference values in YAML strings with {{ metrics.path.to.value }}:

// metrics.json
{
  "summary": {
    "total_assets": 128,
    "categories": 9
  }
}

# presentation.yaml
title: "{{ metrics.summary.total_assets }} Assets"
# → Renders as: "128 Assets"

Core rule: Never hardcode numbers or dynamic values. Always reference metrics.json via interpolation.

Theme & Colors

The theme section fully controls the visual identity. At build time, the framework generates a custom variables.css — all accent variants are derived automatically from your hex values.

# Dark green + gold (luxury brand)
theme:
  primary: "#0D1F0D"
  accent: "#D4AF37"
  fonts:
    heading: "Cormorant Garamond"
    subheading: "Raleway"
    body: "Source Sans Pro"

From the accent hex, the builder derives:

CSS VariableWhat It DoesDerived As
--pf-accentPrimary accent colorExact hex value
--pf-accent-dimDimmed accent (borders)rgba at 0.4 opacity
--pf-accent-glowSubtle glowrgba at 0.15 opacity
--pf-accent-borderBorder colorrgba at 0.3 opacity
--pf-accent-bgBackground tintrgba at 0.1 opacity
--pf-primary-darkDarker backgroundPrimary darkened 60%
--pf-card-borderCard bordersAccent at 0.2 opacity

Fonts are loaded from Google Fonts CDN — use any family name available there.

Layout: title

Centered hero slide with decorative frame, large title, and optional feature grid.

layout: title
data:
  title: "JEMA"                       # Required
  subtitle: "Subtitle"                 # Optional
  tagline: "One-line description"      # Optional
  quote: "\"Quoted text.\""             # Optional (italic)
  title_size: "120px"                  # Optional (default 120px)
  features:                              # Optional — grid of icons
    - icon: gem
      label: "Feature"
      sub: "Description"
  footer: "Author — Institution"       # Optional

Layout: two-column

Two-column grid with header. Each column contains a list of typed content blocks.

layout: two-column
data:
  title: "Slide Title"               # Required
  subtitle: "Optional"                # Optional
  column_ratio: "2fr 1fr"            # Optional — CSS grid cols
  left:                                # Content blocks
    - type: card | solution-box | stat-grid | table | dist-bars | insight | value-prop | takeaway | html
  right:                               # Same types + unit-grid, val-bars
    - type: ...

Block types:

TypeKey Fields
cardicon, title, text, bullets[]
solution-boxbadge, title, icon, items[]
stat-gridcols (2/3/4), stats[{value, label}]
tableheaders[], rows[[]], winner_rows[], footnote
dist-barsbars[{label, width, pct, color_class}]
val-barsitems[{label, lift, width, color}]
insighttext, icon
htmlcontent (raw HTML)

    - # Column 2
      - ...
    - # Column 3
      - ...

Layout: data-table

Two-section layout optimised for benchmark tables. Uses sections list (each section = one column).

layout: data-table
data:
  title: "Benchmarks"
  sections:
    - section_title: "Section A"
      section_icon: "vector-square"
      table:
        headers: ["Model", "Score"]
        rows:
          - ["Model A", "0.95"]
        winner_rows: [0]
        footnote: "Note"
      insight:
        text: "Key finding."

Layout: stat-grid

KPI dashboard — two columns with stat grids, cards, and insights.

layout: stat-grid
data:
  title: "Key Metrics"
  columns:
    - - type: stat-grid
        stats:
          - { value: "128", label: "Total" }
      - type: card
        header: "Details"
        items: ["Item one", "Item two"]

Layout: closing

Thank you / Q&A slide with decorative frame.

layout: closing

Component: Cards

challenge_card

Left-bordered card with icon circle, title, text, and bullet points. Used via type: card in two-column layouts.

type: card
icon: layer-group           # FontAwesome icon name
title: "Card Title"
text: "Body text with <strong>HTML</strong> support."
bullets:
  - "Bullet point"

solution_box

Accent-bordered box with badge, title, and checklist. Used via type: solution-box.

type: solution-box
badge: "Badge Text"
title: "Box Title"
icon: gem

  - "Checklist item"

Custom Layouts

Create templates/layouts/my-layout.html.j2:

{% extends "base.html.j2" %}
{% from "partials/card.html.j2" import challenge_card %}

{% block content %}
{% include "partials/header.html.j2" %}
<!-- Your layout HTML —>
<!-- Access data via: slide.data.title, slide.data.my_field, etc. —>
{% endblock %}

Then reference it in YAML as layout: my-layout.

Architecture

presentation-framework/
├── pf/
│   ├── __init__.py
│   ├── builder.py          # PresentationBuilder class
│   └── cli.py              # Click CLI (init, build, serve, zip)
├── templates/
│   ├── base.html.j2        # HTML shell (head, body, CSS links)
│   ├── present.html.j2     # Navigator with keyboard controls
│   ├── layouts/            # 6 layout templates
│   └── partials/           # Reusable Jinja2 macros
├── theme/
│   ├── variables.css       # Design tokens (stock defaults)
│   ├── base.css            # Slide container, header, grid
│   └── components.css      # Cards, stats, tables, bars, pills
├── setup.py                # pip install -e .
├── SKILL.md                # AI agent instructions
├── README.md
└── docs/
    └── index.html          # This documentation

Key Python API

# Direct Python usage (no CLI)
from pf.builder import PresentationBuilder

builder = PresentationBuilder(
    config_path="presentation.yaml",
    metrics_path="metrics.json"
)
output_path = builder.build(output_dir="slides")
