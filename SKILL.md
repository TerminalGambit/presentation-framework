# Skill: Presentation Framework

## Description
Build branded HTML slide decks from a `presentation.yaml` config + `metrics.json` data file. The framework uses Jinja2 templates with a shared CSS theme to generate individual HTML slides and an iframe-based navigator with keyboard controls.

## Prerequisites
- Python 3.10+
- The `presentation-framework` package installed: `pip install -e /path/to/presentation-framework`
- A `metrics.json` data file (required â€” never hardcode data)

## Quick Start
```bash
# Scaffold a new project
pf init my-deck

# Edit my-deck/presentation.yaml and my-deck/metrics.json

# Build slides (add --open to auto-launch in browser)
pf build --config my-deck/presentation.yaml --metrics my-deck/metrics.json --output my-deck/slides --open

# Preview with live-reload
pf serve --dir my-deck/slides --watch

# Package for sharing
pf zip --dir my-deck/slides

# Export to PDF (requires: pip install pf[pdf] && playwright install chromium)
pf pdf --config my-deck/presentation.yaml --output my-deck/deck.pdf
```

## Agent Workflow

When a user asks you to build a presentation:

1. **Understand the content** â€” Ask what the presentation is about, who the audience is, and what data they have
2. **Structure metrics.json first** â€” Organize all data (numbers, lists, tables) into a clean JSON structure
3. **Plan the slide sequence** â€” Choose layouts that match the content:
   - Start with `title`, end with `closing`
   - Use `section` dividers between major topics
   - Use `two-column` for detailed content with cards, tables, stats
   - Use `data-table` for benchmarks and comparisons
   - Use `stat-grid` for KPI dashboards
   - Use `chart` for interactive data visualization
   - Use `quote`, `image`, `timeline` for variety and pacing
4. **Write presentation.yaml** â€” Reference `{{ metrics.path }}` for all dynamic values
5. **Build and iterate** â€” Fix warnings, adjust content density, verify rendering

### Content Density Guidelines
- **Cards**: 2-3 per column in two-column layout. Each card: title + 1-2 sentences + 2-3 bullets max
- **Tables**: 4-6 rows max per table section
- **Stats**: 2-4 stat boxes per stat-grid
- **Timeline**: 3-5 steps (more gets cramped)
- If a slide has overflow warnings, split it into two slides rather than shrinking content

## Core Rule
**All data must flow from `metrics.json`**. Use `{{ metrics.path.to.value }}` interpolation in YAML strings â€” never hardcode numbers, counts, or dynamic values.

## presentation.yaml Structure
```yaml
meta:
  title: "Deck Title"
  authors: ["Author 1", "Author 2"]
  institution: "School or Company"
  date: "2026-01-01"

theme:
  primary: "#1C2537"      # Background color â€” any hex value
  accent: "#C4A962"       # Accent color â€” any hex value
  fonts:
    heading: "Playfair Display"
    subheading: "Montserrat"
    body: "Lato"

slides:
  - layout: title         # Layout template name
    data:                  # Data passed to the template
      title: "My Title"
      subtitle: "Subtitle"
```

## Theme Customisation
The `theme` section in `presentation.yaml` **fully controls the color scheme**. The builder generates a custom `variables.css` at build time â€” all accent variants (dim, glow, border, background) are automatically derived from the hex colors you provide.

To match a company's brand, just change `primary` and `accent`:
```yaml
# Example: dark green + white-gold for a luxury brand
theme:
  primary: "#0D1F0D"
  accent: "#D4AF37"
  secondary_accent: "#4A90D9"     # Optional second accent
  style_preset: modern             # modern (default), minimal, or bold
  transition: fade                 # Default slide transition (fade/slide/zoom/flip)
  transition_speed: 0.5s           # Transition duration
  fonts:
    heading: "Cormorant Garamond"
    subheading: "Raleway"
    body: "Source Sans Pro"
```
The builder derives all CSS custom properties (`--pf-accent-dim`, `--pf-accent-glow`, `--pf-card-border`, etc.) from the hex accent color, so every component automatically adapts.

Fonts are loaded from Google Fonts CDN â€” use any family name available there.

### Speaker Notes
Add notes to any slide â€” press `N` during presentation to toggle:
```yaml
slides:
  - layout: title
    notes: "Welcome the audience and introduce the team."
    data:
      title: "My Deck"
```

### Per-Slide Transitions
Override the default transition on individual slides:
```yaml
slides:
  - layout: section
    transition: zoom
    data:
      title: "New Chapter"
```

### Math Equations (LaTeX)
Enable KaTeX rendering in `theme`:
```yaml
theme:
  math: true
```

Use `$...$` for inline math and `$$...$$` for display math in any text field:
```yaml
- type: card
  title: "Loss Function"
  text: "Cross-entropy: $L = -\\sum y_i \\log(\\hat{y}_i)$"
```

### Interactive Charts
Enable Plotly.js for the `chart` layout:
```yaml
theme:
  charts: true   # Enable Plotly.js for chart layout
```

## Metrics Interpolation
Any string value in `data:` can reference metrics using `{{ metrics.path.to.value }}`:
```yaml
# metrics.json: {"summary": {"total_assets": 128, "categories": 9}}
slides:
  - layout: title
    data:
      title: "{{ metrics.summary.total_assets }} Assets"
      # Renders as: "128 Assets"
```
Interpolation works inside any string at any nesting depth â€” titles, bullets, table cells, etc.

## Available Layouts

### 1. `title` â€” Opening / hero slide
Centered layout with decorative frame, large title, and optional feature grid.
```yaml
- layout: title
  data:
    title: "JEMA"                     # Required â€” large heading
    subtitle: "Subtitle Here"         # Optional â€” uppercase subheading
    tagline: "One-line description"   # Optional â€” body text
    quote: "\"Quoted text.\""         # Optional â€” italic accent text
    title_size: "120px"               # Optional â€” default 120px
    features:                         # Optional â€” bottom feature grid
      - icon: gem                     # FontAwesome icon name (no "fa-" prefix)
        label: "Feature Label"
        sub: "Description"
    footer: "Author â€” Institution"    # Optional â€” bottom text
```

### 2. `two-column` â€” Main content layout
Two-column grid with header. Each column contains a list of typed content blocks.
```yaml
- layout: two-column
  data:
    title: "Slide Title"             # Required
    subtitle: "Optional subtitle"
    column_ratio: "2fr 1fr"          # Optional â€” CSS grid-template-columns
    left:                            # List of content blocks
      - type: card
        icon: layer-group
        title: "Card Title"
        text: "Card body text."
        bullets:
          - "Bullet point with <strong>HTML</strong> support"
      - type: solution-box
        badge: "Badge Text"
        title: "Box Title"
        icon: gem
        items:
          - "Checklist item 1"
          - "Checklist item 2"
      - type: stat-grid
        cols: 2                      # 2, 3, or 4
        stats:
          - { value: "128", label: "Assets" }
          - { value: "9", label: "Categories" }
      - type: table
        section_title: "Table Section"
        section_icon: "chart-bar"
        headers: ["Col A", "Col B"]
        rows:
          - ["Row 1A", "Row 1B"]
        winner_rows: [0]             # Highlighted row indices
        footnote: "Note text"
      - type: dist-bars
        section_title: "Distribution"
        bars:
          - { label: "Category A", width: 80, pct: "80%", color_class: "bar-gold" }
      - type: insight
        text: "<strong>Key insight</strong> â€” explanation here."
        icon: lightbulb
      - type: value-prop
        title: "Value Title"
        text: "Value description."
      - type: takeaway
        text: "Key takeaway text."
      - type: html
        content: "<div>Raw HTML</div>"
    right:                           # Same block types as left, plus:
      - type: unit-grid
        tiers:
          - { tier: "Basic", price: "$10", unit: "per month" }
      - type: val-bars
        items:
          - { label: "Metric", lift: "+12%", width: 75, color: "#C4A962" }
```

### 3. `three-column` â€” Multi-column content
Three-column grid. Uses `columns` (list of 3 lists) instead of `left`/`right`.
```yaml
- layout: three-column
  data:
    title: "Slide Title"
    column_ratio: "1fr 1fr 1fr"      # Optional
    columns:
      - # Column 1 â€” list of blocks
        - type: card
          header: "Card Header"
          icon: chart-line
          content: "<p>HTML content</p>"
      - # Column 2
        - type: stat-grid
          stats:
            - { value: "42", label: "Things" }
      - # Column 3
        - type: image
          src: "path/to/image.png"
          alt: "Description"
```
Supported block types: `card`, `stat-grid`, `table`, `val-bars`, `insight`, `image`, `html`.

### 4. `data-table` â€” Benchmarks and comparisons
Two-section layout optimized for tables, bars, and analysis. Uses `sections` list.
```yaml
- layout: data-table
  data:
    title: "Benchmark Results"
    subtitle: "Model comparison"
    column_ratio: "1fr 1fr"          # Optional
    sections:
      - section_title: "Section A"
        section_icon: "vector-square"
        table:
          headers: ["Model", "Score", "Notes"]
          rows:
            - ["Model A", "0.95", "Best"]
            - ["Model B", "0.87", "Good"]
          winner_rows: [0]
          total_row: ["Total", "â€”", "â€”"]  # Optional
          footnote: "Footnote text"
        bars:                        # Optional â€” dist_bars
          - { label: "A", width: 95, pct: "95%" }
        unit_tiers:                  # Optional â€” unit_grid
          - { tier: "Tier 1", price: "$50", unit: "per run" }
        insight:
          text: "Key finding."
          icon: lightbulb
      - section_title: "Section B"
        table:
          headers: ["Prompt", "Accuracy"]
          rows:
            - ["v1", "54.2%"]
```

### 5. `stat-grid` â€” KPI / metrics dashboard
Two-column layout with stat grids, cards, and insights.
```yaml
- layout: stat-grid
  data:
    title: "Key Metrics"
    column_ratio: "1fr 1fr"
    columns:
      - # Column 1
        - type: stat-grid
          cols: 2
          stats:
            - { value: "128", label: "Total Assets" }
            - { value: "9", label: "Categories" }
        - type: insight
          text: "Important note."
      - # Column 2
        - type: card
          header: "Details"
          icon: info-circle
          items:                     # Rendered as checklist
            - "Item one"
            - "Item two"
        - type: section-title
          text: "Section"
          icon: chart-bar
```

### 6. `closing` â€” Thank you / Q&A slide
Centered layout with decorative frame (same as title).
```yaml
- layout: closing
  data:
    title: "Thank You"
    subtitle: "Questions & Discussion"
    title_size: "72px"               # Optional â€” default 72px
    info_items:                      # Optional â€” row of pills/info
      - type: pill
        icon: "fa-brands fa-github"  # Full FA class for pills
        text: "github.com/user/repo"
      - type: info
        icon: "fa-solid fa-graduation-cap"
        text: "Institution Name"
    footer: "Footer text"
```

### 7. `image` â€” Full-bleed or split image
```yaml
- layout: image
  data:
    image: "https://example.com/photo.jpg"  # Required â€” image URL or path
    position: full                           # full (default) or split
    title: "Image Title"                     # Optional
    caption: "Photo description"             # Optional
```

### 8. `section` â€” Section divider
```yaml
- layout: section
  data:
    title: "Part Two"           # Required â€” large heading
    subtitle: "Deep Dive"       # Optional
    number: 2                   # Optional â€” section number
```

### 9. `quote` â€” Centered quotation
```yaml
- layout: quote
  data:
    text: "The only way to do great work is to love what you do."  # Required
    author: "Steve Jobs"        # Optional
    role: "Co-founder, Apple"   # Optional
```

### 10. `timeline` â€” Horizontal step visualization
```yaml
- layout: timeline
  data:
    title: "Project Roadmap"    # Required
    steps:                      # Required â€” 2-6 steps
      - icon: search            # FontAwesome icon name
        title: "Research"
        description: "Gather requirements"
      - icon: pencil
        title: "Design"
        description: "Create mockups"
      - icon: code
        title: "Build"
        description: "Implementation"
      - icon: rocket
        title: "Launch"
        description: "Go live"
```

### 11. `chart` â€” Interactive Plotly chart
Renders an interactive Plotly.js chart. Requires `theme.charts: true`.
```yaml
- layout: chart
  data:
    title: "Revenue by Quarter"        # Optional
    subtitle: "FY 2025-2026"           # Optional
    chart_type: bar                     # bar (default), line, pie, scatter
    source: "{{ metrics.revenue }}"     # Metrics ref to object with labels/values
```

The `source` object (or direct `labels`/`values`/`series` fields) should have:
```yaml
# Option A: source as metrics reference (recommended)
# metrics.json: {"revenue": {"labels": ["Q1","Q2","Q3","Q4"], "values": [10,20,30,40]}}
- layout: chart
  data:
    title: "Revenue"
    chart_type: bar
    source: "{{ metrics.revenue }}"

# Option B: inline data
- layout: chart
  data:
    title: "Revenue"
    chart_type: line
    labels: ["Q1", "Q2", "Q3", "Q4"]
    values: [10, 20, 30, 40]

# Option C: multi-series
- layout: chart
  data:
    title: "Comparison"
    chart_type: bar
    labels: ["Q1", "Q2", "Q3", "Q4"]
    series:
      - name: "Product A"
        values: [10, 20, 30, 40]
      - name: "Product B"
        values: [5, 15, 25, 35]
```

## FontAwesome Icons
Icons use FontAwesome 6 Free. In `data:` fields, use just the icon name without prefixes:
```yaml
icon: gem          # Renders as: <i class="fas fa-gem"></i>
icon: chart-line   # Renders as: <i class="fas fa-chart-line"></i>
```
Exception: `closing` layout `info_items` use full FA class strings (e.g., `"fa-brands fa-github"`).

## Custom HTML Blocks
Any layout that accepts content blocks supports `type: html` for raw HTML:
```yaml
- type: html
  content: |
    <div style="padding: 10px; border: 1px solid var(--pf-accent);">
      Custom content here
    </div>
```

## Step-by-Step: Creating a New Presentation
1. **Prepare `metrics.json`** â€” Populate with all data the slides will reference
2. **Write `presentation.yaml`** â€” Define slides using layouts above, referencing `{{ metrics.x.y }}` for all dynamic values. Set `theme.primary` and `theme.accent` to match the brand.
3. **Build** â€” `pf build --config presentation.yaml --metrics metrics.json --output slides --open`
4. **Verify** â€” Browser opens automatically. Use arrow keys to navigate, `G` for grid overview, `F` for fullscreen
5. **Iterate** â€” Edit YAML, rebuild, refresh browser
6. **Share** â€” `pf zip --dir slides` creates a `.zip` file that anyone can extract and open

## Common Pitfalls

| Mistake | Fix |
|---------|-----|
| Hardcoding data in YAML | Put all data in `metrics.json`, reference with `{{ metrics.x }}` |
| Overfilling two-column slides | Max 3 cards per column. Split into multiple slides. |
| Using `columns` in two-column layout | `two-column` uses `left`/`right`. Only `three-column` and `stat-grid` use `columns`. |
| Missing `theme.math: true` | LaTeX won't render without enabling KaTeX |
| Missing `theme.charts: true` | Chart layout needs Plotly enabled |
| Full FA classes in regular layouts | Only `closing` info_items use `"fa-brands fa-github"`. Other layouts use just `icon: github`. |
| `{{ metrics }}` with no metrics.json | Build succeeds but references appear as literal text |
| Huge tables (10+ rows) | Tables overflow. Use 4-6 rows max, or split across sections. |

## MCP Server Tools

When used as an MCP server, the framework exposes these tools:

| Tool | Purpose |
|------|---------|
| `build_presentation(config_path, metrics_path, output_dir)` | Build slides, returns count + warnings |
| `validate_config(config_path)` | Validate YAML against JSON schema |
| `check_contrast(primary, accent, secondary_accent)` | WCAG 2.1 contrast ratio check |
| `list_layouts()` | List all layouts with descriptions |
| `get_layout_example(layout_name)` | Get YAML data shape for a specific layout |
| `init_presentation(name, directory)` | Scaffold a new project |

### Using MCP Tools
```bash
# Start the MCP server
python -m pf.mcp_server
```

Configure in `.mcp.json`:
```json
{
  "mcpServers": {
    "pf": {
      "command": "python",
      "args": ["-m", "pf.mcp_server"]
    }
  }
}
```
