<objective>
Implement data source plugins so developers can create plugins that fetch data from Google Sheets, REST APIs, or databases, and have the fetched data merged into metrics for slide interpolation.

Purpose: Data source plugins eliminate manual data copying. Users declare `datasources:` in presentation.yaml; the builder fetches data at build time and injects it into the metrics dict before `{{ metrics.x }}` interpolation.
Output: Builder datasource resolution, schema update, and comprehensive tests.
</objective>

**pf/builder.py** — add datasource resolution in `build()`:

After `self.load_metrics()` and BEFORE the schema validation / slide rendering loop, add a datasource resolution step:

```python
# Resolve data sources — fetch external data and merge into metrics
datasources = self.config.get("datasources", [])
if datasources and self._registry:
    self._resolve_datasources(datasources)
```

Add a new method `_resolve_datasources(self, datasources: list[dict]) -> None`:

1. Load credentials from environment variables and optionally from `.pf/credentials.json` in the config file's parent directory (if it exists). Merge env credentials over file credentials.
   ```python
   import os
   creds = {}
   creds_file = self.config_path.parent / ".pf" / "credentials.json"
   if creds_file.exists():
       with open(creds_file) as f:
           creds = json.load(f)
   # Env vars override file credentials
   for key, value in os.environ.items():
       if key.startswith("PF_"):
           creds[key.lower()] = value
   ```

2. For each datasource entry in the list:
   ```yaml
   # Expected YAML structure:
   datasources:
     - plugin: sheets          # registered datasource name
       config:                 # passed to fetch()
         sheet_id: "abc123"
         worksheet: "Sheet1"
       merge_key: "sheets_data"  # key under which result is merged into metrics
   ```

3. Look up the plugin: `ds_plugin = self._registry.get_datasource(entry["plugin"])`. If not found, emit a Click warning and continue.

4. Call `result = ds_plugin.fetch_fn(entry.get("config", {}), creds)`. Wrap in try/except:
   - `PluginCredentialError` -> `click.echo(style("error: ...", fg="red"), err=True)` with the error message. Raise `click.ClickException` to halt the build (missing credentials = cannot continue).
   - Other `Exception` -> warning, continue without this datasource.

5. Merge result into `self.metrics` under the `merge_key`:
   ```python
   merge_key = entry.get("merge_key", entry["plugin"])
   self.metrics[merge_key] = result
   ```

Import `from pf.registry import PluginCredentialError` at the top of builder.py (alongside existing PluginRegistry import).
