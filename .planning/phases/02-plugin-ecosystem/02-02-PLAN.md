<objective>
Implement layout plugin template inheritance and CSS isolation so that plugin layouts can extend core layouts and ship their own scoped CSS without leaking styles into core slides.

Purpose: Without CSS isolation, installing a plugin risks breaking existing presentations. Without template inheritance, plugins must duplicate entire base layouts instead of just overriding blocks.
Output: Updated base template with plugin CSS links, builder CSS copy logic, and tests for inheritance and isolation.
</objective>

**pf/builder.py** — add plugin CSS copy logic in `build()`:

After the existing `shutil.copytree(THEME_DIR, theme_out)` line, add:

1. Create `plugins_css_dir = theme_out / "plugins"`.
2. Iterate `self._registry._layouts.values()`. For each plugin that is a `LayoutPlugin` (not `LocalLayoutPlugin`), iterate its `css_files`. For each CSS file path, if it exists, copy to `plugins_css_dir / f"{plugin.name}.css"` (creating the dir first with `mkdir(parents=True, exist_ok=True)`).
3. For `LocalLayoutPlugin`, check for a CSS file at `plugin.template_path.parent / f"{plugin.name}.css"`. If it exists, copy it to `plugins_css_dir / f"{plugin.name}.css"`.
4. Collect the list of plugin CSS filenames (relative paths like `theme/plugins/kanban.css`). Pass this list as `plugin_css_files` to both `render_slide()` and `render_navigator()` template contexts.

Modify `render_slide()` to accept and pass `plugin_css: list[str] | None = None` to the template context.

In the `build()` method, collect plugin CSS paths before the render loop and pass them to each `render_slide()` call.

**templates/base.html.j2** — add plugin CSS links:

After the existing `<link rel="stylesheet" href="theme/components.css"/>` line, add:
```html
{% if plugin_css %}
{% for css_path in plugin_css %}
  <link rel="stylesheet" href="{{ css_path }}"/>
{% endfor %}
{% endif %}
```

This injects `<link>` tags for each plugin CSS file. The `plugin_css` variable defaults to empty list if not provided, so existing non-plugin builds are unaffected.
