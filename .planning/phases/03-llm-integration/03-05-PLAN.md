**Step 1: Create pf/accessibility.py** — Accessibility auditing module.

Use `re` (stdlib) for HTML scanning rather than BeautifulSoup to avoid adding a dependency. The checker scans built HTML files for common accessibility issues.

Define `AccessibilityWarning` dataclass:
```python
@dataclass
class AccessibilityWarning:
    file: str           # e.g., "slide_01.html"
    element: str        # e.g., "<img src='photo.jpg'>"
    issue: str          # e.g., "missing alt attribute"
    suggestion: str     # e.g., "Add alt='photo' based on filename"
    severity: str       # "error" or "warning"
```

Define `check_accessibility(html: str, filename: str = "") -> list[AccessibilityWarning]`:
1. **Missing alt on images**: Use regex `<img[^>]*>` to find all img tags. For each, check if `alt=` attribute exists. If missing:
   - Extract `src` attribute value
   - Generate filename-derived fallback: extract filename from src path, strip extension, replace hyphens/underscores with spaces, title case. E.g., `"images/team-photo.jpg"` -> suggestion `"Add alt='Team Photo'"`
   - Create warning with severity="error"

2. **Missing aria-label on interactive elements**: Use regex to find `<button`, `<a ` (not `<a>` with visible text), `<input`, `<select` tags. For each:
   - Check if `aria-label=` exists OR if the element has inner text content (between opening and closing tags)
   - If neither: create warning with severity="warning", suggestion to add aria-label

3. **Missing role on slide container**: Check if `role="region"` exists on the slide container div. If missing: create warning.

4. Return the list of warnings sorted by severity (errors first).

Define `check_slide_dir(output_dir: str) -> list[AccessibilityWarning]`:
- Scan all `slide_*.html` files in the output directory
- Call `check_accessibility()` on each
- Return aggregated list

Define `generate_alt_text(src: str) -> str`:
- Pure filename-based: extract filename, strip extension, replace `-_` with spaces, title case
- E.g., `"images/quarterly-revenue-chart.png"` -> `"Quarterly Revenue Chart"`
- This is the Phase 3 scope per research recommendation (NOT vision-based)

**Step 2: Update templates/base.html.j2** — Read the file first, then add ARIA attributes.

Add to the main slide container div:
- `role="region"`
- `aria-label="{{ slide_title }}"`
- `tabindex="0"` (make slide focusable for keyboard navigation)

These attributes go on the outermost `<div class="slide-container"` element. Do NOT break existing template structure — just add attributes to the existing div tag.

**Step 1: Add high-contrast mode to theme/base.css** — Read the file first, then append a `.pf-high-contrast` block at the end:

```css
/* ── High-Contrast Mode ──────────────────────────────────────── */
/* Toggle via: document.querySelector('.slide-container').classList.toggle('pf-high-contrast') */
.slide-container.pf-high-contrast {
  --pf-primary: #000000;
  --pf-primary-dark: #000000;
  --pf-accent: #FFD700;
  --pf-white: #FFFFFF;
  --pf-text: #FFFFFF;
  --pf-text-light: #FFFFFF;
  --pf-text-muted: #E0E0E0;
  --pf-text-dim: #CCCCCC;
  --pf-card-bg: rgba(255, 255, 255, 0.1);
  --pf-card-border: rgba(255, 215, 0, 0.5);
  --pf-shadow-card: none;
  background-color: #000000 !important;
}

.slide-container.pf-high-contrast * {
  text-shadow: none !important;
}
```

This provides WCAG AAA contrast (white text on black, gold accent on black). The class can be toggled via JavaScript at runtime.
