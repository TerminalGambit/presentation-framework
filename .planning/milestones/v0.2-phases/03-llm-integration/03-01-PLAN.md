<objective>
Create per-layout Pydantic models in `pf/llm_schemas.py` with cardinality constraints (maxItems on lists, maxLength on strings) so LLMs producing structured output cannot generate slides that overflow.

Purpose: LLM-01 requires that each layout has a documented JSON Schema with constraints. These Pydantic models serve as the `response_model` for `instructor` in Plan 03, and are also exposed via a `get_layout_schema()` MCP tool.

Output: `pf/llm_schemas.py` module with 11 layout models + `PresentationOutput` wrapper, and comprehensive tests.
</objective>

<task type="auto">
  <name>Task 1: Create pf/llm_schemas.py with per-layout Pydantic models</name>
  <files>pf/llm_schemas.py</files>
  <action>
Create `pf/llm_schemas.py` with Pydantic v2 BaseModel classes for all 11 layouts plus a PresentationOutput wrapper.

Use `Field(max_length=N)` on list fields to generate `maxItems` in JSON Schema, and `Field(max_length=N)` on str fields to generate `maxLength`. Derive constraint values from LayoutAnalyzer.SIZE_MODEL and USABLE_HEIGHT (575px):

**Block models (nested, reused across layouts):**
- `CardBlock`: type="card", title(max_length=60), text(max_length=200), bullets(list[str], max_length=4), icon(optional)
- `SolutionBoxBlock`: type="solution-box", badge, title(max_length=60), items(list[str], max_length=4), icon(optional)
- `StatItem`: value(max_length=20), label(max_length=40)
- `StatGridBlock`: type="stat-grid", cols(default=2), stats(list[StatItem], max_length=6)
- `TableBlock`: type="table", headers(list[str]), rows(list[list[str]], max_length=8), winner_rows(optional), footnote(optional)
- `DistBarBlock`: type="dist-bars", bars(list with max_length=8)
- `ValBarBlock`: type="val-bars", items(list, max_length=6)
- `InsightBlock`: type="insight", text(max_length=300), icon(optional)
- `ContentBlock`: Union of all block types above (discriminated union on "type" field)

**Layout models:**
- `TitleSlide`: layout="title"(frozen), title(max_length=80), subtitle(max_length=120), tagline(max_length=200), features(list, max_length=4), footer(max_length=100)
- `TwoColumnSlide`: layout="two-column"(frozen), title(max_length=80), subtitle(max_length=120), left(list[ContentBlock], max_length=3), right(list[ContentBlock], max_length=3)
- `ThreeColumnSlide`: layout="three-column"(frozen), title(max_length=80), columns(list[list[ContentBlock]], max_length=3, with each inner list max_length=2)
- `DataTableSlide`: layout="data-table"(frozen), title(max_length=80), subtitle(max_length=120), sections(list, max_length=2)
- `StatGridSlide`: layout="stat-grid"(frozen), title(max_length=80), columns(list[list[ContentBlock]], max_length=2)
- `ChartSlide`: layout="chart"(frozen), title(max_length=80), subtitle(max_length=120), chart_type(Literal["bar","line","pie","scatter"]), labels(list[str], max_length=12), values(list[float], max_length=12)
- `ClosingSlide`: layout="closing"(frozen), title(max_length=60), subtitle(max_length=120), info_items(list, max_length=4), footer(max_length=100)
- `ImageSlide`: layout="image"(frozen), image(str), position(Literal["full","split"]), title(max_length=80), caption(max_length=200)
- `SectionSlide`: layout="section"(frozen), title(max_length=60), subtitle(max_length=120), number(optional int)
- `QuoteSlide`: layout="quote"(frozen), text(max_length=300), author(max_length=60), role(max_length=80)
- `TimelineSlide`: layout="timeline"(frozen), title(max_length=80), steps(list, min_length=2, max_length=6)

**Wrapper model:**
- `PresentationOutput`: yaml_config(str, description="Full presentation.yaml content"), metrics(dict, description="metrics.json data as dict")

**Lookup functions:**
- `get_layout_schema(layout_name: str) -> type[BaseModel] | None` — returns the model class for a layout name
- `get_all_schemas() -> dict[str, type[BaseModel]]` — returns dict of all layout name -> model class mappings

Use `Literal` types for layout names (frozen fields). Ensure all optional fields have sensible defaults. Do NOT use deprecated `conlist()` — use `Field(max_length=N)` on list fields.

Verify that `TwoColumnSlide.model_json_schema()` produces `"maxItems": 3` on the left/right fields and `TimelineSlide.model_json_schema()` produces `"maxItems": 6` and `"minItems": 2` on the steps field.
  </action>

  <done>All 11 layout Pydantic models exist with cardinality constraints. model_json_schema() produces maxItems/maxLength. get_layout_schema() and get_all_schemas() work.</done>
