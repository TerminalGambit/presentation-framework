**Add `check_accessibility_output` MCP tool:**

```python
@mcp.tool()
def check_accessibility_output(output_dir: str) -> dict:
    """Audit built slide HTML files for accessibility issues.

    Scans all slide_*.html files in the output directory for missing alt
    attributes, missing ARIA labels, and other a11y problems.
    Returns a dict with "warnings" list and "pass" boolean.
    """
```

Implementation:
1. Lazy import `check_slide_dir` from `pf.accessibility`.
2. Validate that `output_dir` exists as a directory (`Path(output_dir).is_dir()`), return `{"error": "..."}` if not.
3. Call `warnings = check_slide_dir(output_dir)`.
4. Convert each `AccessibilityWarning` dataclass to a dict: `[{"file": w.file, "element": w.element, "issue": w.issue, "suggestion": w.suggestion, "severity": w.severity} for w in warnings]`.
5. Return `{"pass": len(warnings) == 0, "warning_count": len(warnings), "warnings": warning_dicts}`.
6. On exception: return `{"error": str(e)}`.

This tool allows AI agents to verify accessibility after `build_presentation()` as part of a generation workflow, completing the LLM-06 requirement.
