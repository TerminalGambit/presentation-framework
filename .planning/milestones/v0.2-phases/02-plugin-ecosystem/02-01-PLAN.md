<task type="auto">
  <name>Task 1: Create pf/registry.py with PluginRegistry, dataclasses, and discovery logic</name>
  <files>pf/registry.py</files>
  <action>
Create `pf/registry.py` containing:

1. **Custom exception**: `PluginCredentialError(Exception)` — raised by data source plugins when credentials are missing.

2. **Plugin dataclasses** (using `@dataclass`):
   - `LayoutPlugin(name: str, description: str = "", templates_dir: Path | None = None, css_files: list[Path] = field(default_factory=list), version: str = "0.0.0")` — the contract for pip-installable layout plugins.
   - `LocalLayoutPlugin(name: str, template_path: Path)` — auto-created for templates found in a local `layouts/` directory.
   - `ThemePlugin(name: str, description: str = "", defaults: dict = field(default_factory=dict), css_file: Path | None = None, version: str = "0.0.0")` — theme plugin contract.
   - `DataSourcePlugin(name: str, description: str = "", fetch_fn: callable = None, version: str = "0.0.0")` — wraps a `fetch(config, credentials) -> dict` callable.

3. **PluginRegistry class**:
   - `__init__`: Initialize empty dicts `_layouts`, `_themes`, `_datasources`.
   - `discover(project_dir: Path | None = None)`: Call `_discover_layouts`, `_discover_themes`, `_discover_datasources`. This is called once at builder startup.
   - `_discover_layouts(project_dir)`:
     - Use `importlib.metadata.entry_points(group="pf.layouts")` to find pip-installed layout plugins. Wrap each `ep.load()` in try/except, emit `warnings.warn()` on failure.
     - If `project_dir` is provided, scan `project_dir / "layouts"` for `*.html.j2` files. Create `LocalLayoutPlugin` for each. The name is the stem with `.html` stripped (e.g., `kanban.html.j2` -> `kanban`).
     - Warn if a local layout name collides with an entry-point layout name.
   - `_discover_themes(project_dir)`:
     - Use `entry_points(group="pf.themes")` similarly.
     - If `project_dir`, scan `project_dir / "themes"` for `*.css` files as local themes.
   - `_discover_datasources()`:
     - Use `entry_points(group="pf.datasources")`. The loaded value is expected to be a callable `fetch(config, credentials) -> dict`. Wrap in `DataSourcePlugin`.
   - Properties: `layout_names -> list[str]`, `theme_names -> list[str]`, `datasource_names -> list[str]`.
   - `get_layout(name) -> LayoutPlugin | LocalLayoutPlugin | None`
   - `get_theme(name) -> ThemePlugin | None`
   - `get_datasource(name) -> DataSourcePlugin | None`
   - `get_template_loader(core_templates_dir: Path) -> ChoiceLoader`:
     - Build list of `FileSystemLoader`s: one per plugin's `templates_dir` (if set), then core loader last.
     - For `LocalLayoutPlugin`s, add a loader for `template_path.parent`.
     - Return `ChoiceLoader(loaders)` where core is always last (plugins can override or add templates).

Use `from __future__ import annotations` for forward references. Import `from importlib.metadata import entry_points`. Do NOT import `pkg_resources`.
