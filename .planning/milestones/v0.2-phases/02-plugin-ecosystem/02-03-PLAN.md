**pf/builder.py** — add theme merge logic:

1. In `build()`, BEFORE `generate_variables_css(theme_cfg)` is called, add theme plugin merge:

```python
# Theme plugin merge: base defaults < user overrides
theme_cfg = self.config.get("theme", {})
theme_name = theme_cfg.get("name")
if theme_name and self._registry:
    base_theme = self._registry.get_theme(theme_name)
    if base_theme:
        # Deep merge: plugin defaults, then user values override
        merged = {**base_theme.defaults}
        # Merge fonts separately (nested dict)
        if "fonts" in base_theme.defaults and "fonts" in theme_cfg:
            merged["fonts"] = {**base_theme.defaults.get("fonts", {}), **theme_cfg["fonts"]}
        merged.update({k: v for k, v in theme_cfg.items() if k != "fonts" or "fonts" not in base_theme.defaults})
        theme_cfg = merged
```

This merge happens inline in build(). The `theme_cfg` variable is then passed to `generate_variables_css()` as before. No changes to `generate_variables_css()` itself.

2. Also copy theme plugin CSS if `base_theme.css_file` exists — add to the `theme/plugins/` directory alongside layout plugin CSS (reuse the same `plugins_css_dir` from Plan 02 if it exists, or create it).

**Important**: The `theme.name` key is NOT in the current schema. Add `"name": {"type": "string"}` to the `theme` properties in `pf/schema.json` so it passes validation.
