**`pf plugins` group:**

def plugins():

**`pf plugins list` command:**
- Create a `PluginRegistry`, call `discover()` (with no project_dir — we want system-wide installed plugins).
- Display three sections: Layouts, Themes, Data Sources.
- For each section, list plugin names sorted alphabetically. If a plugin has a description, show it.
- If a section is empty, show "(none installed)".
- Format:
  ```
  Installed Plugins

    Layouts (2):
      - kanban        Kanban board layout
      - timeline-v2   Custom timeline variant

    Themes (1):
      - corporate     Corporate blue branding

    Data Sources (0):
      (none installed)
  ```
- Use `click.echo()` and `click.style()` for formatting (bold header, consistent indentation).

**`pf plugins install` command:**
- Takes a `package` argument (string).
- Use `sys.executable` to find the correct Python interpreter: `subprocess.run([sys.executable, "-m", "pip", "install", package], capture_output=True, text=True)`. This avoids the "wrong pip" pitfall from research.
- On success: green message `Installed {package}`.
- On failure: red error message with stderr, exit code 1.
- Import `sys` and `subprocess` at the function level (not module level) to keep imports clean.

**`pf plugins info` command:**
- Takes a `plugin_name` argument.
- Create registry, discover.
- Search across all plugin types (layouts, themes, datasources) for a match.
- If found, display: name, type (layout/theme/datasource), version, description, templates_dir (for layouts), defaults (for themes).
- If not found, echo "Plugin '{name}' not found. Run 'pf plugins list' to see installed plugins." and exit 1.
