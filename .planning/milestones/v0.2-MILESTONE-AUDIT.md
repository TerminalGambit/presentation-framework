## Cross-Phase Integration

### Wired: 28 integration paths confirmed

All core data flows verified: builder → templates, registry → builder, CLI → builder, MCP → registry, MCP → optimizer, MCP → sanitize, MCP → accessibility, platform API → worker → builder, analytics → SQLite, WebSocket → ConnectionManager.

### Integration Issues: 3

**1. Beacon/Sync Script Injection Gap (Phase 4)**

`BEACON_SCRIPT` (analytics.py) and `SYNC_SCRIPT` (sync.py) are defined as module-level constants with correct JavaScript implementations. However, `pf_platform/worker.py:run_build()` does not inject either script into the served `present.html` after `builder.build()` completes. The served deck is vanilla HTML with no analytics pings and no WebSocket client.

- Affects: PLAT-03, PLAT-04
- Severity: Tech debt — infrastructure complete, wiring step missing
- Fix: Post-process `present.html` in `run_build()` to inject `<script>` tags with `window.__PF_ANALYTICS_URL`, `window.__PF_DECK_ID`, and `window.__PF_SYNC_URL`

**2. Phase 1 Layouts Missing from LLM Layer (Phase 1 → Phase 3)**

`LAYOUT_DESCRIPTIONS`, `LAYOUT_EXAMPLES` (mcp_server.py), and `_LAYOUT_REGISTRY` (llm_schemas.py) contain only the original 11 layouts. The 5 layouts added in Phase 1 (`code`, `mermaid`, `video`, `map`, `toc`) are absent from all LLM-facing surfaces:
- `generate_presentation` system prompt only lists 11 layouts
- `suggest_layout` constructs available_layouts from `LAYOUT_DESCRIPTIONS.keys()` (11)
- `get_layout_schema` cannot return schemas for the 5 new layouts

- Affects: LLM-01, LLM-02, LLM-03 (cross-phase completeness)
- Severity: Tech debt — LLM tools work for 11/16 layouts
- Fix: Add Pydantic models for the 5 new layouts; update `LAYOUT_DESCRIPTIONS` and `LAYOUT_EXAMPLES`

### Broken E2E Flows: 2

**Flow: Analytics Tracking**
```
POST /api/build → run_build() → present.html (NO beacon) → user views → no events fired → empty dashboard
```
Missing: Beacon script injection in worker.py after build completes.

**Flow: WebSocket Presenter Sync**
```
POST /api/build → present.html (NO sync client) → __PF_SYNC_URL undefined → IIFE guard returns → no WS connection
```
Missing: Sync script injection in worker.py after build completes.
